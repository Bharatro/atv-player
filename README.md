# atv-player

基于 `PySide6` 和 `mpv` 的 `alist-tvbox` 桌面播放器，当前以 Linux 为优先目标平台，同时保留 macOS 和 Windows 打包支持。

应用默认连接 `http://127.0.0.1:4567`，围绕 `alist-tvbox` 后端提供登录、媒体浏览、豆瓣电影、电报影视、B站、网络直播、Emby、Jellyfin、飞牛影视、盘搜、播放记录，以及独立播放器窗口。

## 功能概览

### 登录

- 输入后端地址、用户名和密码进行登录
- 自动记住上次使用的后端地址和用户名
- 登录成功后自动获取并缓存 `vod token`
- 不保存密码

### 主界面

- **标签页**：根据后端能力自动显示以下标签页
  - 豆瓣电影
  - 电报影视
  - B站
  - 网络直播
  - Emby
  - Jellyfin
  - 飞牛影视
  - 盘搜
  - 文件浏览
  - 播放记录
- **插件标签页**：为每个启用的 `TvBox Python` 爬虫插件生成独立标签页
- **插件溢出抽屉**：当标签页过多时，隐藏的插件收入"更多"抽屉，支持搜索
- **插件右键菜单**：重新加载、编辑名称、编辑配置、启用/禁用
- **全局搜索栏**：在所有已启用的标签页中同时搜索
  - 显示各来源结果数量，如"豆瓣电影(23)"
  - 支持直接打开：磁力链接（离线下载）、网盘分享链接（阿里、夸克、百度、PikPak等）、HTTP URL（内置解析器）
- **启动恢复**：自动恢复上次播放会话，包括占位会话处理
- 顶部提供插件管理、直播源管理、退出登录按钮

### 豆瓣电影 / 电报影视 / 飞牛影视

- 海报卡片网格展示，自动响应窗口宽度（1-6列）
- 左侧分类列表，支持关键词搜索
- 筛选面板：按年份、地区、类型等过滤
- 分页浏览（上一页/下一页）
- 点击海报进入详情或浏览

### B站

- 分类浏览：番剧、电影、电视剧等
- 文件夹层级导航，支持面包屑
- 详情页显示投币、点赞、收藏、回复、弹幕数量
- DASH 播放，支持清晰度选择
- 外挂字幕支持
- 本地播放历史记录

### Emby / Jellyfin

- 媒体库浏览：电影、剧集、收藏
- 文件夹层级导航
- 直接流式播放
- 向服务器报告播放进度
- 本地历史恢复

### 盘搜

- 跨多个网盘搜索资源
- 搜索结果直接打开浏览

### 文件浏览

- 面包屑导航（首页 + 路径段，可点击跳转）
- 表格列：类型、名称、大小、豆瓣ID、评分、时间
- 列头点击排序（类型列除外）
- 文件类型图标：文件、文件夹、视频、音频、文本、图片、播放列表
- 内联搜索面板，支持按网盘类型筛选（百度、天翼、夸克、UC、阿里、115、123、迅雷、移动、PikPak）
- 分页浏览，可选每页 20/30/50/100 条
- 记住每个路径的分页状态

### 播放记录

- 表格列：标题、集数、当前播放、进度、时间、来源
- 来源标签：插件、Emby、B站、Jellyfin、飞牛影视、全局解析、远程
- 双击恢复播放
- 删除选中记录或清空全部
- 分页浏览

### 播放器

使用独立播放器窗口播放视频。

#### 播放控制

- 播放/暂停（Space）
- 进度条拖动，悬浮显示时间提示
- 快退/快进按钮（15秒、60秒）
- 上一集/下一集（PgUp/PgDn）
- 音量滑块，静音切换（M）
- 倍速选择：0.5x / 0.75x / 1.0x / 1.25x / 1.5x / 2.0x
- 片头/片尾跳过时长设置（数值输入框）
- 重新播放按钮

#### 显示模式

- 全屏（Enter 或双击视频）
- 宽屏模式（W），隐藏侧边栏扩大视频区域
- 返回主窗口（Ctrl+P 或 Esc）

#### 播放列表与来源

- 播放列表面板：显示当前来源的所有集数
- 来源分组选择（多组播放列表时）
- 线路选择（多线路时）
- 双击列表项切换集数
- 自动播放下一集

#### 详情面板

- 海报显示（180x260）
- 详情操作按钮（投币、点赞、收藏等，来源相关）
- 详情字段展示，支持链接点击
- 可收起的播放日志面板

#### 字幕

- 主字幕选择：自动选择 / 关闭 / 手动选择轨道或外挂字幕
- 次字幕选择：关闭 / 手动选择轨道或外挂字幕
- 右键菜单调整主字幕/次字幕的位置和大小
  - 位置预设：顶部、偏上、默认、偏下、底部
  - 大小预设：很小、小、默认、大、很大
  - 微调：上移/下移 5%、缩小/放大 5%、重置
- ASS 字幕覆盖模式支持
- 外挂字幕文件自动加载（.ass/.srt）

#### 音轨

- 音轨选择：自动选择 / 手动选择
- 显示编码、声道数、采样率信息

#### 清晰度

- DASH 视频清晰度选择
- 显示分辨率、编码、码率信息

#### 弹幕

**弹幕来源**：

- 支持 5 个弹幕提供方：腾讯、优酷、B站、爱奇艺、芒果
- 智能搜索匹配：
  - 标题模糊匹配与相似度评分
  - 自动集数识别
  - 时长过滤（±5分钟）
  - 电影变体检测（原声版、普通话版、粤语版）
  - 过滤预告、花絮、特辑等补充内容
  - 来源 URL 自动匹配优先提供方
- 多提供方并发搜索（4线程）
- 搜索结果按提供方分组显示
- 记住每个系列的弹幕来源偏好

**弹幕源对话框**（D 键）：

- 可编辑媒体标题和集数
- 搜索来源选择：全部 / 腾讯 / 优酷 / B站 / 爱奇艺 / 芒果
- 提供方列表 + 选项列表双栏展示
- 重新搜索、恢复默认搜索词、切换并加载

**弹幕设置对话框**（Ctrl+D 键）：

- 显示行数：1-10
- 显示模式：静态 / 仅滚动 / 混合
- 位置预设：顶部 / 顶部偏下 / 中上 / 底部
- 颜色模式：统一颜色 / 保留原色
- 统一颜色：颜色选择器
- 文字大小：16-72px（步进2）
- 滚动速率：0.5x - 2.0x（步进0.1）
- 恢复默认按钮

**弹幕右键菜单**：

- 弹幕配置：开启 / 关闭
- 弹幕源
- 弹幕设置

**缓存**：

- 搜索缓存：7天有效期
- XML 缓存：文件缓存
- ASS 缓存：版本化缓存（含渲染参数）

#### 视频信息

- I 键显示视频信息（编码、分辨率、码率等）

#### 恢复播放

- 自动恢复上次播放位置
- 根据 URL 或集数索引匹配播放列表中的位置
- 定时向后端上报进度（每5秒）
- 本地播放历史保存

#### 右键菜单

播放器视频区域右键菜单：

- 主字幕（选择轨道/外挂）
- 次字幕（选择轨道/外挂）
- 主字幕位置（预设 + 微调）
- 次字幕位置（预设 + 微调）
- 主字幕大小（预设 + 微调）
- 次字幕大小（预设 + 微调）
- 音轨（选择轨道）
- 清晰度（DASH 视频质量选择）
- 弹幕配置（开启/关闭）
- 弹幕源
- 弹幕设置
- 视频信息

### 网络直播

#### 直播源管理

- 内置一个默认远程直播源
- 支持三类直播源：
  - **远程 M3U**：输入 M3U URL
  - **本地 M3U**：选择本地 M3U 文件
  - **手动维护**：逐条添加频道
- 每个来源支持：启用/禁用、重命名、删除、刷新
- 表格显示：名称、类型（远程/本地/手动）、地址、启用状态、刷新状态、最近刷新时间

#### EPG 电子节目指南

- 支持 EPG URL 配置（多行，每行一个）
- 立即更新按钮
- 刷新状态显示

#### 手动直播源

- 频道管理表格：分组、频道名、地址、Logo
- 添加/编辑/删除频道
- 上移/下移调整顺序

#### 频道特性

- 解析 `tvg-logo`、`group-title` 和 `http-header` / `http-user-agent` 等 M3U 属性
- 会合并同组下同名频道，把多线路组织成一个播放列表

### 插件

- 支持本地和远程 `TvBox Python` 爬虫插件
- 支持插件启用/禁用、重命名、上移/下移、刷新、删除、查看加载日志
- 远程插件会缓存到本地后再加载
- 支持 GitHub 导入（显示进度和结果摘要：新增/更新/跳过）
- 插件排序对话框：置顶/上移/下移/置底
- 每个插件支持自定义配置文本
- 插件提供自定义动作按钮（显示在插件管理器中）

> 远程插件会执行本地 Python 代码，只应加载受信任来源。

### HLS 代理

内置本地 HTTP 代理（默认 `127.0.0.1:2323`），用于：

- M3U8 播放列表重写，去除广告片段（`/adjump/`、`/ad-`、`/ad/` 等）
- TS 分片缓存（60秒 TTL，最多 200 片）与预取（提前获取后续 2 片）
- TS 分片修复：移除 PNG 前缀、修复 TS 包对齐（188字节同步）
- DASH 清单解析与质量选择
- ISO 蓝光光盘流式播放（HTTP range 请求 + UDF 文件系统解析）
- 会话管理（900秒 TTL，令牌认证）

### 蓝光 ISO 播放

- 远程 ISO 文件的 HTTP range 读取
- UDF 文件系统解析
- 主播放列表自动检测（最大 m2ts）
- MPLS/CLPI 解析
- 片头裁剪（每片段 5 分钟，总计 10 分钟）
- 循环播放列表检测
- 重叠播放列表检测
- 分片组合与缓存

### 内置解析器

支持多种播放解析器：虾米、fish、jx1、jx2、mg1、tx1 等，可设置偏好的解析器。

### 卡拉OK

- 支持三种歌词格式：QQ 音乐 QRC、酷狗 KRC、网易 YRC
- 字级时间同步
- ASS 渲染：`\kf` 卡拉OK效果，翻译行显示

## 环境要求

- Python `3.12+`
- `uv`
- 可用的 `libmpv`
- 一个可访问的 `alist-tvbox` 后端

Linux 上如果系统里没有 `libmpv`，运行和打包都会失败。`build.py` 会在常见系统目录查找它。

## 快速开始

安装开发依赖：

```bash
uv sync --group dev
```

启动应用：

```bash
./start.sh
```

`start.sh` 实际执行的是：

```bash
uv run src/atv_player/main.py
```

## 本地数据

应用使用 `Qt` 的标准数据目录和缓存目录。Linux 上通常分别是：

```text
~/.local/share/atv-player
~/.cache/atv-player
```

主要文件和目录：

- 配置数据库：`~/.local/share/atv-player/app.db`
- 插件缓存：`~/.local/share/atv-player/plugins/cache`
- 海报缓存：`~/.cache/atv-player/posters`
- 弹幕搜索缓存：`~/.local/share/atv-player/danmaku-search-cache.json`
- 弹幕系列偏好：`~/.local/share/atv-player/danmaku-series-preferences.json`

应用通过本地 `sqlite` 保存以下状态：

- 后端地址、用户名、登录令牌和 `vod token`
- 最近浏览路径
- 上次活跃窗口、上次活跃标签页和分类
- 上次播放来源、模式、路径、`vod_id` 和恢复信息
- 播放器音量、静音状态、宽屏模式、日志可见性
- 主窗口、浏览页、播放器窗口布局状态和分隔条位置
- 直播源、手动频道和 EPG 配置
- 插件配置和加载日志
- 弹幕偏好（启用、行数、显示模式、颜色模式、颜色、位置、速率、字号）
- 偏好的解析器

应用不会保存密码。

## 开发

运行测试：

```bash
uv run pytest
```

运行 `ruff`：

```bash
uv run ruff check .
```

项目采用 `src` 布局，主要目录如下：

- `src/atv_player`：应用代码
  - `controllers/`：各标签页控制器（豆瓣、电报、B站、Emby、Jellyfin、飞牛、盘搜、直播、播放、历史、登录）
  - `ui/`：界面组件（主窗口、播放器窗口、登录窗口、各对话框）
  - `player/`：播放器核心（mpv 封装、恢复逻辑、蓝光 ISO、M3U8 广告过滤）
  - `danmaku/`：弹幕系统（服务、缓存、5个提供方、字幕渲染、偏好）
  - `karaoke/`：卡拉OK（QRC/KRC/YRC 解析、ASS 渲染）
  - `proxy/`：HLS 代理（服务器、M3U8 重写、分片缓存、广告过滤、DASH 支持）
  - `plugins/`：插件系统（加载器、仓库、兼容层、加密运行时）
- `tests`：单元测试和 UI 测试
- `packaging`：各平台图标和打包资源
- `docs/superpowers`：设计说明与实现计划

## 打包

本地打包和 GitHub Actions 共用同一个入口：

```bash
uv sync --group dev --group package
uv run python build.py current
```

也可以显式指定目标平台：

```bash
uv run python build.py linux
uv run python build.py macos
uv run python build.py windows
```

各平台输出规则：

- Linux：先生成 `PyInstaller` 目录包，再封装成 `AppImage`
- macOS：生成 `.app`
- Windows：生成单文件 `.exe`

Linux 打包额外要求：

- 系统里需要 `appimagetool`
- 系统里需要可用的 `libmpv`

Windows 打包时，`build.py` 会优先从这些位置查找 `mpv` 运行库：

- 环境变量 `ATV_MPV_RUNTIME_DIR`
- 仓库根目录下的 `mpv/`
- 当前 `PATH`

GitHub Actions 会为 Pull Request 和手动触发构建 Linux、macOS、Windows 制品；推送以 `v` 开头的标签时，还会创建 GitHub Release 并上传产物。
