# 追更详情分集虚拟卡片网格设计

## 概要

将“我的追更”详情页中的分集区保留为当前的完整卡片视觉，但把底层实现从“每集一个真实 QWidget 卡片”改成“虚拟化网格卡片绘制”。

目标：

- 保留当前卡片式浏览体验，不回退成列表行。
- 保持 `1 / 2 / 3` 列切换能力。
- 支持单季超长内容，例如 1200 集动画。
- 切季、切列、图片异步加载时不再创建并保留 1200 个分集 widget。

## 变更范围

主要涉及：

- `src/atv_player/ui/following_episode_browser.py`
- `tests/test_following_episode_browser.py`
- `tests/test_following_detail_page_ui.py`

不涉及：

- `FollowingController` 详情抓取逻辑
- `FollowingEpisode` / `FollowingDetailSnapshot` 数据结构
- 分集预览弹窗的数据来源和交互
- 页面顶部元数据区、海报轮播区、演职员区的大布局重做

## 现状问题

当前分集区虽然已经有 model/view 结构，但右侧卡片区仍然使用：

- `QScrollArea`
- `QGridLayout`
- 每集一个 `FollowingEpisodeCard` widget

这会导致：

- 1200 集时一次性创建 1200 个 QWidget
- 布局计算和刷新成本随集数线性膨胀
- 切换列数时需要全量重排和重建卡片
- 缩略图返回时需要逐个真实卡片刷新

根因不是卡片外观复杂，而是“卡片以 widget 形式全量存在”。

## 设计目标

### 功能目标

- 用户仍然看到完整卡片，而不是列表行
- 用户仍可在 `1 / 2 / 3` 列之间切换
- 用户点击卡片后继续打开现有分集预览弹窗
- 切季后仍保留当前的选中项恢复和滚动恢复逻辑

### 性能目标

- 当前季即使有 1200 集，也只绘制可视区域附近卡片
- 切换列数只更新虚拟网格参数和重绘，不重建 1200 个 widget
- 缩略图异步返回时只刷新对应 index，不触发布局级全量抖动

### 视觉目标

- 保持现有卡片的信息结构：
  - 16:9 封面
  - 标题
  - 状态 badge
  - 日期 / 时长
  - 简介摘要
- 三档列数只是密度变化，不是内容模式变化

## 总体方案

继续使用 `QListView + QAbstractListModel + QStyledItemDelegate`，但把右侧分集区从“纵向列表行”改成“虚拟化网格卡片”。

推荐实现：

- `QListView` 使用网格/图标式布局能力承载卡片项
- `EpisodeListModel` 继续提供一维分集数据
- `EpisodeItemDelegate` 负责绘制完整卡片视觉
- `1 / 2 / 3` 列切换只改变网格尺寸、卡片尺寸、文字布局和摘要行数

不再使用：

- `QScrollArea + QGridLayout + FollowingEpisodeCard` 作为分集主渲染路径

## 右侧分集区布局

右侧分集区保持现有结构：

- 顶部工具条
- 下方分集卡片区

工具条继续保留单按钮列数切换：

- `1 列`
- `2 列`
- `3 列`

变化点仅在下方内容区：

- 从真实 widget 网格改成虚拟化卡片网格

## 卡片信息结构

三档列数都必须保留完整卡片结构：

- 封面
- 标题
- 状态 badge
- 日期 / 时长
- 简介摘要

禁止退化成：

- 只有标题和日期的列表行
- 去掉封面
- 去掉状态 badge

## 三档列数规则

### 1 列

- 大封面
- 标题最多 2 行
- 状态 badge 保留在标题区
- 日期 / 时长保留
- 摘要最多 3 行

用途：

- 最完整的浏览模式

### 2 列

- 封面缩小一档
- 标题最多 2 行
- 状态 badge 保留
- 日期 / 时长保留
- 摘要最多 2 行

用途：

- 默认推荐密度
- 在完整信息和滚动效率之间平衡

### 3 列

- 封面再缩小一档
- 标题最多 2 行
- 状态 badge 保留，但可缩小内边距和字号
- 日期 / 时长保留
- 摘要最多 2 行

用途：

- 高密度浏览模式
- 仍保持“卡片感”，不变成列表

## 交互规则

- 单击卡片：打开现有分集预览弹窗
- 切季：替换当前季数据，保留已有的选中项和滚动恢复策略
- 切列：只更新卡片网格参数和重绘
- 键盘导航：继续由 view 提供

## 缩略图加载

继续复用 `EpisodeThumbnailStore`：

- delegate 在无图时绘制占位封面
- store 返回图片后，仅触发对应 index 的重绘
- 季简介区海报仍可继续复用同一 store

## 数据与状态

`EpisodeListModel` 继续承载：

- 当前季分集列表
- 当前进度
- 当前季号
- 最新集状态
- 下一集状态

新增或调整：

- 列数映射到卡片绘制参数，而不是映射到“列表/网格双实现”
- delegate 根据当前列数读取对应卡片布局参数

## 实现边界

本次不做：

- 新的筛选、排序、分页
- 区间分段浏览
- 新的弹窗或预览交互
- 新的配置项语义

现有 `following_episode_grid_columns` 仍作为列数配置沿用。

## 测试策略

### 浏览器单测

覆盖：

- `1 / 2 / 3` 列切换时只更新虚拟卡片网格参数
- 大季数据下不再创建 1200 个 `FollowingEpisodeCard`
- 状态文本和当前集状态仍正确暴露
- 缩略图返回时只刷新对应项

### 详情页 UI 测试

覆盖：

- 打开详情页仍能显示分集区
- 切季后仍能切换当前季内容
- 点击分集仍能打开预览弹窗
- 现有进度与状态接线不回退

## 验收标准

- 视觉上仍然是完整卡片，不是列表行
- `1 / 2 / 3` 列切换都保留封面、标题、badge、日期 / 时长、摘要
- 进入 1200 集详情时不再因为全量创建分集 widget 导致明显卡顿
- 现有分集点击、切季、状态显示行为保持不变
