# Danmaku Search Title Size Suffix Design

## Summary

Strip trailing file-size suffixes such as `(1.74 GB)` from the default danmaku search title generated by `SpiderPluginController`.

This change is intentionally narrow. It fixes playback items whose displayed title is decorated by the backend with a size suffix and prevents that suffix from polluting danmaku search queries such as `你好星期六 20250104期(1.74 GB)`.

## Goals

- Remove backend-added trailing size suffixes from the default danmaku search title.
- Preserve valid episode information in variety-style calendar titles such as `20250104期`.
- Keep the existing year-suffix stripping behavior.
- Limit behavior changes to the default danmaku search title path.

## Non-Goals

- Changing provider-side danmaku search behavior.
- Changing generic `normalize_name()` behavior used across the wider danmaku package.
- Removing non-size technical filename suffixes in this change.
- Changing manually overridden danmaku search queries.

## Design

The change stays inside `src/atv_player/plugins/controller.py`.

- Add a small helper that strips a trailing parenthesized file-size suffix.
- Support common backend formats such as `KB`, `MB`, `GB`, and `TB`, with optional decimal values and optional whitespace.
- Apply the helper when resolving the default danmaku search title from `media_title` or `title`.
- Keep the existing trailing-year cleanup in the same path, so both suffix cleanups can apply together.
- Only affect the default search title resolution path. If the user explicitly overrides the danmaku search title or full query, that value is preserved.

## Examples

- `你好星期六 20250104期(1.74 GB)` -> `你好星期六 20250104期`
- `12(894.9 MB)` -> `12`
- `黑夜告白 (2026)(2.0 GB)` -> `黑夜告白`

## Testing

- Add a controller test proving the default search title drops a trailing `GB` suffix while keeping the calendar episode text.
- Add a controller test proving a trailing `MB` suffix is also removed.
- Keep existing year-suffix tests passing to confirm the cleanup order still works.
