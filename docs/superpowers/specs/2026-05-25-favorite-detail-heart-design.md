# 收藏按钮标题内嵌设计

## 概要

本次调整只修改收藏入口的 UI 表达，不改收藏仓储、控制器、记录结构或收藏页行为。

目标：

- 播放页把收藏按钮移动到“影片详情”标题文字后面，保持少量水平间距。
- 收藏按钮为无边框图标按钮。
- 未收藏显示灰色空心心形，已收藏显示红色实心心形。
- 列表页右键菜单对未收藏项目显示“加入收藏”，对已收藏项目显示“取消收藏”。

## 变更范围

涉及模块：

- `src/atv_player/ui/player_window.py`
- `src/atv_player/ui/browse_page.py`
- `tests/test_player_window_ui.py`
- `tests/test_browse_page_ui.py`

不涉及模块：

- 收藏仓储与控制器
- 收藏页面卡片列表
- 首页收藏入口
- 主窗口收藏路由

## 播放页设计

### 标题区布局

当前收藏按钮位于详情动作区之前的独立 action widget 中。改造后，按钮将和“影片详情”标题处于同一行：

- 使用标题行容器承载 `metadata_heading`
- 在标题 label 后面紧接收藏按钮
- 按钮与标题之间保留一个固定的小间距
- 保持详情动作区仅用于现有 detail actions

这样可以让收藏入口视觉上附着在详情标题本身，而不是被理解为通用操作按钮区的一部分。

### 按钮样式与状态

按钮继续使用现有收藏切换回调，但视觉语义改为：

- 未收藏：灰色空心心形
- 已收藏：红色实心心形
- 无边框
- icon-only

同时保留可访问性和悬浮提示：

- 未收藏 tooltip：`加入收藏`
- 已收藏 tooltip：`取消收藏`
- accessible name 与 tooltip 保持一致

### 状态刷新

按钮状态仍由当前播放项驱动：

- 打开会话时刷新
- 当前播放项变化时刷新
- 点击切换后立即刷新

本次不改收藏判定逻辑，只改 `refresh` 后呈现出的图标、颜色和位置。

## 列表页右键菜单设计

浏览列表视频项右键菜单继续复用当前收藏切换逻辑，不新增第二套收藏入口。

文案统一为：

- 未收藏：`加入收藏`
- 已收藏：`取消收藏`

这只影响文案，不影响触发行为、上下文菜单结构或项对象解析逻辑。

## 风险与兼容性

主要风险：

- 播放页标题行布局调整可能影响现有详情区 spacing
- 收藏按钮移位后，原有测试对 widget 层级或容器可见性的假设可能失效

兼容策略：

- 不删除现有收藏切换回调，只重新挂载到标题行
- 不修改 `favorite_is_active` / `favorite_toggle` 接口
- 不改右键菜单项的触发路径，只改展示文案

先写或调整失败测试，再实现代码。

- 未收藏时 tooltip 为“加入收藏”，已收藏时为“取消收藏”
