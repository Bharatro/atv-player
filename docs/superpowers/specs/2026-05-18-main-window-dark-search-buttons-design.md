# Main Window Dark Search Buttons Design

## Summary

优化主窗口深色主题下的全局搜索区，让 `搜索按钮` 和 `热搜词按钮` 在不改布局的前提下更容易识别。

本次改动只增强两个圆形图标按钮的视觉层级，不调整主窗口顶部布局、按钮数量、交互语义或搜索流程。

## Goals

- 提升深色主题下 `搜索按钮` 与 `热搜词按钮` 的可见性
- 保持现有搜索输入框与两个按钮的整体布局不变
- 让两个按钮在视觉上比输入框更像独立操作入口
- 保持浅色主题和其他主窗口控件的现有行为稳定

## Non-Goals

- 不新增文字标签、气泡提示或额外入口
- 不修改按钮尺寸、位置、间距或 header 布局结构
- 不改动全局搜索弹窗、热搜加载、搜索状态流转等业务逻辑
- 不重做主窗口顶部其他按钮的风格

## Scope

主要修改文件：

- `src/atv_player/ui/main_window.py`
- `src/atv_player/ui/theme.py`
- `src/atv_player/ui/icon_cache.py`

## Current Problem

当前主窗口搜索区使用深色输入框加两个 36x36 的圆形图标按钮。深色主题下存在两个问题：

- 图标资源默认描边偏暗，和深色背景对比不足
- 按钮背景与输入框背景接近，两个操作入口没有从输入框中明显分离出来

结果是用户在深色主题下更难快速识别 `搜索` 和 `热搜词` 两个动作。

## Chosen Approach

采用“对比增强”方案：

- 两个按钮继续保留现有圆形纯图标样式
- 图标改为按当前主题 token 着色，深色主题下使用更亮的前景色
- 按钮背景相对输入框提升一个层级
- 按钮边框对比增强，`hover` 状态进一步提升可见性
- 可以增加很轻的阴影或内描边，但不能做成强调色主按钮

这个方案比“轻量增强”更有效，又比“强调色增强”更克制，不会抢走顶部其他操作按钮的视觉权重。

## Styling Rules

深色主题下两个按钮应满足以下视觉规则：

- 默认图标颜色明显亮于当前 SVG 原始描边
- 默认按钮底色比 `global_search_edit` 的输入框背景更亮一级
- 默认边框对比度高于当前实现，和输入框边界可区分
- `hover` 状态下背景或边框继续抬升一层，形成明确反馈
- 禁用态继续沿用统一 disabled token，不做额外特殊处理

浅色主题下不追求明显变化，应继续保持和现有设计接近，避免引入不必要回归。

## Implementation Notes

`MainWindow` 当前直接通过 `load_icon(...)` 加载 `search.svg` 与 `rank.svg`。由于 SVG 内部带固定描边色，本次应避免继续直接依赖原始颜色。

推荐实现方向：

- 在 `icon_cache.py` 提供基于原图缓存再着色的辅助函数
- `MainWindow` 根据 `current_tokens()` 为两个按钮加载亮色图标
- `theme.py` 中扩展或调整圆形图标按钮样式构建函数，让按钮背景、边框和 hover 更适合深色主题

这样可以把“图标亮度”和“按钮容器层级”分开控制，后续如果其他深色主题图标按钮需要同类增强，也能复用。

## Testing Strategy

在 `tests/test_main_window_ui.py` 增加或扩展主窗口 UI 回归测试，至少覆盖：

- 深色主题下搜索按钮和热搜词按钮不是空图标
- 深色主题下两个按钮使用增强后的圆形按钮样式，样式中包含更强边框或更高层级背景 token
- 深色主题下两个按钮的图标经过主题着色，而不是直接依赖原始 SVG 默认颜色
- 现有搜索区布局约束保持不变，例如按钮尺寸和搜索容器宽度不变

测试应尽量锚定可稳定断言的样式 token、按钮属性或图标像素差异，避免依赖脆弱的完整 QSS 字符串顺序。

## Risks

- 仅靠 QSS 不能改变 SVG 内固定描边色，因此如果不加运行时着色，视觉提升会有限
- 图标着色测试如果直接断言整张 pixmap 的完整二进制内容，可能受平台渲染差异影响，需要选更稳健的比较方式
- 若按钮背景提升过头，会让搜索区两个操作按钮显得比右侧主工具按钮更重，需要控制层级幅度
