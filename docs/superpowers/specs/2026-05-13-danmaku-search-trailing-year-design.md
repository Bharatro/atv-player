# 弹幕搜索标题尾部年份清洗设计

## 概要

当播放项的默认弹幕搜索标题来自媒体标题时，去掉标题末尾仅用于展示的年份后缀，例如 `黑夜告白 (2026)` 改为 `黑夜告白`。这次调整只影响默认搜索标题生成，不修改用户手动输入，也不改 provider 侧的通用名称规范化逻辑。

## 目标

- 默认弹幕搜索标题不再携带尾部括号年份。
- 只处理尾部年份后缀，不误伤片名中的数字。
- 不改变手动覆盖搜索标题的行为。

## 非目标

- 不在 `normalize_name(...)` 或 provider 搜索逻辑里做全局年份清洗。
- 不处理标题中间位置的年份文本。

## 规则

- 仅清洗标题末尾的 4 位年份后缀。
- 支持 `(...)`、`（...）`、`[...]`、`【...】` 这类包裹形式。
- 年份前后的空白允许存在。
- 如果去掉后标题为空，则回退到原值。

示例：

- `黑夜告白 (2026)` -> `黑夜告白`
- `黑夜告白（2026）` -> `黑夜告白`
- `黑夜告白 [2026]` -> `黑夜告白`
- `歌手2026` -> 保持不变

## 落点

- 在 `src/atv_player/plugins/controller.py` 的默认弹幕搜索标题解析路径处理。
- 在 `tests/test_spider_plugin_controller.py` 增加回归测试。
