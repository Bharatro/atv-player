# 列表页视频右键菜单设计

## 概要

为主窗口所有海报列表页卡片增加统一的右键菜单入口，并按来源类型与当前页面状态控制显示项。

本轮只处理菜单展示，不实现新的收藏能力，也不增加详情弹窗。

目标菜单项：

- `打开播放`
- `全局搜索`
- `加入收藏`

本轮明确不加入：

- `显示详情`

## 目标

- 为所有 `PosterGridPage` 卡片增加右键菜单能力。
- 将菜单显示规则集中在 `MainWindow`，避免页面组件耦合业务来源判断。
- 对 `豆瓣电影`、`网络直播`、`全局搜索结果页` 应用不同的菜单可见性规则。
- 为后续真实“加入收藏”功能预留稳定入口。

## 非目标

- 本轮不实现真正的收藏持久化、收藏列表、收藏管理或收藏提示。
- 本轮不增加 `显示详情` 菜单项。
- 本轮不修改左键点击行为。
- 本轮不为历史记录页或其他非海报列表页增加同类菜单。

## 当前问题

当前海报列表页卡片只有左键点击行为：

- `PosterGridPage` 只在点击时发出 `search_requested`、`open_requested`、`item_open_requested`
- 不支持右键菜单
- 各来源页面也没有统一的列表项二级操作入口

因此用户无法在列表页直接执行“打开播放 / 全局搜索 / 加入收藏”这类补充动作。

## 方案比较

### 方案一：在 `PosterGridPage` 内直接写菜单规则

做法：

- 页面组件自己判断当前是不是豆瓣、直播、全局搜索结果
- 直接创建并显示最终菜单

优点：

- 初看实现最短

缺点：

- `PosterGridPage` 会知道太多业务来源语义
- 后续再加规则时会继续膨胀
- 不利于插件页和主窗口状态统一处理

### 方案二：`PosterGridPage` 只发出右键请求，`MainWindow` 生成菜单

做法：

- `PosterGridPage` 给卡片加自定义右键菜单支持
- 页面只把当前 `VodItem` 和触发位置抛给外层
- `MainWindow` 根据页面实例和当前状态构造菜单

优点：

- 业务规则集中
- 页面组件保持通用
- 更适合后续继续扩展菜单项

缺点：

- 需要增加一层信号转发

### 方案三：抽象完整菜单描述模型

做法：

- 先建立一套通用菜单 DSL 或数据模型
- 页面和主窗口都围绕模型工作

优点：

- 可扩展性最好

缺点：

- 对当前需求明显过度设计

## 选型

采用方案二。

这次需求只是给现有卡片补一个统一菜单入口，但菜单规则依赖页面来源和 `MainWindow` 中的全局搜索状态。把规则集中在 `MainWindow`，可以保持 `PosterGridPage` 轻量，同时不给后续菜单扩展制造额外耦合。

## 设计

### 1. 右键触发链路

`PosterGridPage` 增加卡片级右键菜单支持：

- 为卡片按钮设置自定义右键菜单策略
- 右键时发出包含当前 `VodItem`、卡片按钮、局部坐标或全局坐标的信号
- `PosterGridPage` 不直接决定显示哪些业务动作

`MainWindow` 负责：

- 连接各个 `PosterGridPage` 的右键信号
- 根据触发页面和当前状态创建 `QMenu`
- 执行对应动作

### 2. 菜单项集合

本轮实际菜单只包含以下三类动作：

- `打开播放`
- `全局搜索`
- `加入收藏`

`显示详情` 本轮不创建，不保留占位项，也不显示禁用态。

### 3. 可见性规则

基础规则：

- 普通列表页默认显示 `打开播放`、`全局搜索`、`加入收藏`

特殊规则：

- `豆瓣电影`
  - 不显示 `打开播放`
  - 不显示 `加入收藏`
  - 显示 `全局搜索`
- `网络直播`
  - 显示 `打开播放`
  - 不显示 `全局搜索`
  - 显示 `加入收藏`
- `全局搜索结果页`
  - 不显示 `全局搜索`
  - 其他项仍按来源规则保留

组合结果举例：

- 豆瓣电影普通分类页：只显示 `全局搜索`
- 网络直播全局搜索结果页：显示 `打开播放`、`加入收藏`
- 普通来源全局搜索结果页：显示 `打开播放`、`加入收藏`
- 普通来源普通分类页：显示 `打开播放`、`全局搜索`、`加入收藏`

### 4. 动作行为

#### `打开播放`

行为与当前左键打开逻辑一致：

- 走现有来源对应的打开链路
- 继续复用 `item_open_requested` / 现有 `MainWindow` 来源处理方法

对 `豆瓣电影` 不提供此动作，因为豆瓣列表本身只支持搜索跳转，不支持直接播放。

#### `全局搜索`

行为：

- 使用当前卡片标题填充主窗口全局搜索输入框
- 触发现有 `_start_global_search()` 流程

该动作在以下场景不显示：

- 当前页面已经处于全局搜索结果模式
- 当前来源是 `网络直播`

#### `加入收藏`

本轮仅提供入口，不实现真实收藏能力。

行为定义：

- 菜单点击后走 `MainWindow` 中的单独占位处理函数
- 该处理函数暂不写数据库、不改配置、不刷新页面数据
- 需要保留清晰的方法边界，方便后续接入真实收藏仓库

对 `豆瓣电影` 不显示此动作。

### 5. 来源识别方式

来源规则由 `MainWindow` 根据页面实例识别：

- `self.douban_page` 视为 `豆瓣电影`
- `self.live_page` 视为 `网络直播`
- 其他 `PosterGridPage` 视为普通来源页

是否处于“全局搜索结果页”使用 `PosterGridPage` 的外部结果状态识别，而不是单纯依赖当前 tab 标题文案。

### 6. 后续扩展边界

为了让本轮改动能承接后续功能，菜单构造和动作分发应保持以下边界：

- `PosterGridPage` 只负责卡片右键事件
- `MainWindow` 负责业务规则和动作映射
- `加入收藏` 使用独立方法入口，后续可以直接替换为真实收藏服务

## 错误处理

- 如果右键时没有有效 `VodItem`，不显示菜单。
- 如果根据当前规则计算后没有任何菜单项，不显示空菜单。
- `加入收藏` 占位动作本轮不抛出业务错误，也不修改现有播放/搜索状态。
